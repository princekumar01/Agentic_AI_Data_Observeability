# 🏥 Clinical Trial AI Data Observability System

**Agentic AI-powered Data Observability POC for Clinical Trials**

Monitors clinical trial data pipelines across **5 observability pillars** using LangGraph multi-agent orchestration, with full PII/PHI masking, audit trail, and human-in-the-loop review.

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| UI | Streamlit 1.35 |
| Backend | FastAPI 0.111 + Uvicorn |
| Agent Orchestration | LangGraph 0.1.9 |
| LLM | HuggingFace (Llama-3.1-8B via LangChain-OpenAI) |
| Data Processing | Pandas + SciPy (KS-test drift) |
| PII/PHI Masking | Microsoft Presidio + spaCy |
| Visualization | Plotly |
| Audit Trail | File-based JSON (one per run) |

## Setup & Installation

### 1. Prerequisites and Repository Setup
Ensure you have Python 3.9+ installed. If you haven't already, navigate to the project directory.

### 2. Create and activate a virtual environment (using uv)

```bash
uv venv
source .venv/bin/activate        # macOS/Linux
.venv\Scripts\activate           # Windows
```

### 3. Install dependencies

```bash
uv pip install -r requirements.txt
```

### 4. Download the spaCy NLP model (required by Presidio)

```bash
python -m spacy download en_core_web_lg
```
*Note: If that fails, or you prefer a much faster, ~50MB model (fine for PII work), use `python -m spacy download en_core_web_sm` instead.*

### 5. Set up your environment variables

```bash
cp .env.example .env
```

Edit `.env` and add your HuggingFace API key:

```
HF_API_KEY=hf_your_actual_key_here
HF_TOKEN=hf_your_actual_key_here
LLM_MODEL=meta-llama/Llama-3.1-8B-Instruct
```

### 6. Place the CSV file

Copy the provided sample dataset from the `Data_Sources/` folder to `data/clinical/`:

```bash
mkdir -p data/clinical
cp Data_Sources/clinical_trial_data_observability.csv data/clinical/clinical_trial_data.csv
```

The CSV must have these columns:
`patient_id, patient_name, age, medication, blood_pressure, glucose_level, side_effect, severity, visit_date, hospital_name`

### 7. Generate baseline metrics (one-time)

```bash
python scripts/setup_baseline.py
```

This reads your CSV and writes `config/baseline_metrics.json` for drift detection.

## Running the Application

Open **two terminals** from the project root:

**Terminal 1 — FastAPI Backend:**
```bash
uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
```

**Terminal 2 — Streamlit Frontend:**
```bash
streamlit run frontend/app.py --server.port 8501
```

Open your browser at: **http://localhost:8501**

## Application Flow

```
1. Go to Page 1 (Pipeline) → Click "Run Pipeline"
2. Watch live progress as pipeline executes:
   - Input Discovery → Entry Validation → Preprocessing
   - ETL Log Generation → PII/PHI Masking → 5 AI Agents
3. When complete, go to Page 2 (Review)
   - Read the AI-generated incident report
   - Enter your Reviewer ID and click "Approve"
4. Dashboard (Page 3) unlocks automatically
   - View Plotly charts, agent findings, audit trail
   - Download the approved incident report
```

## Observability Pillars

| Pillar | Implementation |
|--------|---------------|
| 🕐 **Freshness** | File modified-at check + visit_date recency validation |
| 📦 **Volume** | Row count vs expected, delta % alert |
| 🗂 **Schema** | Column presence, dtype verification, null rates, duplicate IDs |
| 📈 **Distribution** | IQR outlier detection + KS-test drift vs baseline |
| 🔗 **Lineage** | ETL log correlation — errors/warnings mapped to anomalies by RCA agent |

## Output Files Per Run

Each run creates a folder at `output/runs/{run_id}/`:

| File | Description |
|------|-------------|
| `audit_trail.json` | Complete immutable audit log — all prompts, responses, decisions |
| `etl_run.log` | Raw ETL execution log generated during preprocessing |
| `metrics.json` | Raw Pandas metrics output |
| `sanitized_metrics.json` | PII-masked metrics (what agents see) |
| `sanitized_log.txt` | PII-masked ETL log (what log agent sees) |
| `incident_report_draft.md` | AI-generated draft (before approval) |
| `incident_report.md` | Final approved report (only exists after HITL approval) |

## Troubleshooting

**`HF_API_KEY not found`** → Check your `.env` file has `HF_API_KEY=hf_...`

**`CSV file not found`** → Place the CSV at `data/clinical/clinical_trial_data.csv`

**`spaCy model not found`** → Run `python -m spacy download en_core_web_lg`

**`Connection refused on port 8000`** → Make sure the FastAPI backend is running

**LLM timeout** → The HuggingFace model may be cold-starting. Retry after 30 seconds.
