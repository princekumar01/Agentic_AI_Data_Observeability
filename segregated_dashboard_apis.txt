4.GET/seg-dashboard/run/{run_id}/null-rate

why:Returns null percentage per column with the 5% threshold. Powers the Null Rate by Column bar chart. Columns exceeding the threshold are highlighted red.

Response body
Download
[
  {
    "column": "patient_id",
    "null_pct": 0,
    "threshold": 5
  },
  {
    "column": "age",
    "null_pct": 0,
    "threshold": 5
  },
  {
    "column": "medication",
    "null_pct": 0.2,
    "threshold": 5
  },
  {
    "column": "blood_pressure",
    "null_pct": 0.8,
    "threshold": 5
  },
  {
    "column": "glucose_level",
    "null_pct": 2.4,
    "threshold": 5
  },
  {
    "column": "side_effect",
    "null_pct": 7.4,
    "threshold": 5
  },
  {
    "column": "severity",
    "null_pct": 0,
    "threshold": 5
  },
  {
    "column": "visit_date",
    "null_pct": 0,
    "threshold": 5
  },
  {
    "column": "hospital_name",
    "null_pct": 0,
    "threshold": 5
  }
]
______________________________________additional info___________________________
Here null_pct is different from the previous two — it is not calculated from other values in this response. It is calculated directly from the raw patient data.

The math — how backend calculates null_pct:
null_pct = (number of null/empty cells in that column / total records) × 100
For side_effect column:
37 null cells / 500 total records × 100 = 7.4%
For glucose_level column:
12 null cells / 500 total records × 100 = 2.4%
For patient_id column:
0 null cells / 500 total records × 100 = 0%

5.GET /seg-dashboard/run/{run_id}/severity-distribution

why:Returns anomaly counts broken down by severity (Critical, High, Medium, Low). Powers the Severity Distribution donut chart.

[
  {
    "severity": "Critical",
    "count": 3,
    "pct": 17.6,
    "color": "#EF4444"
  },
  {
    "severity": "High",
    "count": 5,
    "pct": 29.4,
    "color": "#F59E0B"
  },
  {
    "severity": "Medium",
    "count": 7,
    "pct": 41.2,
    "color": "#3B82F6"
  },
  {
    "severity": "Low",
    "count": 2,
    "pct": 11.8,
    "color": "#10B981"
  }
]
________________________________________additional info_________________
The math:
Total anomalies = 3 + 5 + 7 + 2 = 17
Critical  →  3  / 17 × 100 = 17.6%
High      →  5  / 17 × 100 = 29.4%
Medium    →  7  / 17 × 100 = 41.2%
Low       →  2  / 17 × 100 = 11.8%
                              ──────
                              100%

8.GET/seg-dashboard/run/{run_id}/token-usage
why:Returns total tokens (input/output), estimated cost, and per-agent breakdown. Powers the Token Usage section — run total card + agents bar chart.

	
Response body
Download
{
  "run_total_input": 24810,
  "run_total_output": 8340,
  "run_total": 33150,
  "estimated_cost": 0.0281,
  "model": "claude-sonnet-4",
  "by_agent": [
    {
      "agent": "Data Quality Agent",
      "tokens": 12580,
      "pct": 37.9
    },
    {
      "agent": "Log Analysis Agent",
      "tokens": 6820,
      "pct": 20.6
    },
    {
      "agent": "Root Cause Agent",
      "tokens": 5640,
      "pct": 17
    },
    {
      "agent": "Recommendation Agent",
      "tokens": 4520,
      "pct": 13.6
    },
    {
      "agent": "Compliance Agent",
      "tokens": 3590,
      "pct": 10.8
    }
  ]
}



_________________________additional info______________________________
pct = percentage share of the total tokens that each agent consumed.

The math — how backend calculates it:
pct = (agent_tokens / run_total) × 100
For Data Quality Agent:
12580 / 33150 × 100 = 37.9%
For Compliance Agent:
3590 / 33150 × 100 = 10.8%
All 5 pct values add up to 100% (37.9 + 20.6 + 17 + 13.6 + 10.8 = 100).
